# Tool-Integrated Reasoning (TIR)

A small-scale replication of the [TORA paper](https://arxiv.org/abs/2309.17452) (ICLR 2024), comparing **Chain-of-Thought (CoT)** reasoning against **Tool-Integrated Reasoning (TIR)** for mathematical problem solving. Where CoT models must perform all computation internally via natural language, TIR models interleave natural language rationale with executable Python code, offloading arithmetic and symbolic manipulation to a code interpreter.

## Goal

Replicate the core finding of the TORA paper — that TIR outperforms rationale-only CoT — using a deliberately small setup: **Qwen2.5-0.5B** fine-tuned on just 1,000 training examples from GSM8k. The aim is not to match TORA's absolute numbers (which used LLaMA-2 7B–70B and 16k GPT-4-generated examples), but to validate that the TIR advantage holds even at minimal scale.

## Method

### Models

- **Base model:** [Qwen2.5-0.5B](https://huggingface.co/Qwen/Qwen2.5-0.5B) (494M parameters)
- **Data generation model:** [Qwen2.5-Math-7B-Instruct](https://huggingface.co/Qwen/Qwen2.5-Math-7B-Instruct) (already TIR-capable natively)

### Training Data

**CoT** — 1,000 examples sampled from the GSM8k train split. Each example pairs a math problem with its ground-truth rationale-only solution (natural language chain of thought ending in `#### <answer>`).

**TIR** — 1,000 examples from the same GSM8k problems, with trajectories generated by Qwen2.5-Math-7B-Instruct. Each trajectory interleaves natural language reasoning with Python code blocks and their execution outputs, following the TORA format:

````
<rationale>
```python
<code>
```
```output
<execution result>
```
<rationale with \\boxed{answer}>
````

Of the 1,000 generated trajectories, 963 produced correct answers and were used for training.

### Training Setup

Both models were trained from the same Qwen2.5-0.5B checkpoint with identical hyperparameters:

| Parameter | Value |
|---|---|
| Epochs | 1 |
| Batch size | 2 |
| Gradient accumulation | 4 (effective batch 8) |
| Learning rate | 2e-5 (cosine decay) |
| Warmup | 3% of steps |
| Max sequence length | 2048 |

Loss is computed only on the response tokens (the question/prompt is masked).

### Evaluation

Both models were evaluated on 200 examples from the GSM8k test split using greedy decoding. The CoT model generates rationale-only solutions; the TIR model generates interleaved rationale + code, with code blocks executed in a sandboxed interpreter (up to 3 tool-use rounds, 5-second timeout per execution).

### Key Deviations from TORA

| Aspect | TORA | This Project |
|---|---|---|
| Base model | LLaMA-2 7B–70B | Qwen2.5-0.5B |
| Training data | 16k GPT-4-generated examples | 1,000 Qwen2.5-Math-7B-Instruct examples |
| Output space shaping | Yes (64 samples/problem, teacher correction) | Skipped |
| Eval set size | Full test sets | 200 examples (GSM8k) |

## Results

### GSM8k (n_train=1000, epochs=1)

| Model | Correct | Total | Accuracy |
|---|---|---|---|
| CoT | 60 | 200 | **30.0%** |
| TIR | 77 | 200 | **38.5%** |
| **Delta (TIR - CoT)** | | | **+8.5%** |

TIR achieves a **28% relative improvement** over CoT on the same model, training data size, and hyperparameters.

### Example Outputs

**TIR** (correct) — the model writes and executes code to compute the answer:

> *Problem: Marie ordered one chicken meal that costs \$12, 5 packs of milk that costs \$3 each, 4 apples that cost \$1.50 each, and some boxes of pizza. Marie paid a total of \$50. How many boxes of pizza did Marie order if each box costs \$8.50?*
>
> The model sets up variables for each item, computes total known costs in Python, subtracts from \$50, and divides by \$8.50 to get `2.0`. Final answer: `\boxed{2}`.

**CoT** (correct) — the model reasons entirely in natural language:

> *Problem: Tracy used a piece of wire 4 feet long to support tomato plants. The wire was cut into pieces 6 inches long. How many pieces did she obtain?*
>
> "There are 12 inches in 1 foot, so 4 feet is 4×12 = 48 inches. The wire was cut into pieces 6 inches long, so Tracy obtained 48/6 = 8 pieces. `#### 8`"

## Usage

### Run a full experiment (train both models + evaluate + compare)

```bash
python scripts/run_experiment.py \
    --dataset gsm8k \
    --n 1000 \
    --tir_data_path data/tir_gsm8k_1000.jsonl \
    --output_dir results/exp_gsm8k_n1000 \
    --eval_limit 200
```

### Generate TIR training data

```bash
python scripts/generate_local.py --dataset gsm8k --n 1000
```

### Train individual models

```bash
# CoT baseline
python scripts/train_cot.py --dataset gsm8k --epochs 1 --output_dir checkpoints/cot_gsm8k

# TIR
python scripts/train_tir.py --data_path data/tir_gsm8k_1000.jsonl --epochs 1 --output_dir checkpoints/tir_gsm8k
```

### Evaluate

```bash
# CoT
python scripts/evaluate.py --model checkpoints/cot_gsm8k/final --dataset gsm8k --prompt_format cot --answer_format native

# TIR
python scripts/evaluate.py --model checkpoints/tir_gsm8k/final --dataset gsm8k --prompt_format tir --answer_format boxed
```
